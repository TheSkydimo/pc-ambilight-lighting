## 先选路线：你要“PC 专用”还是“信号源通吃”

### 路线 A：PC 软件直抓（简单、PC 专用）

- **适合**：Windows/PC、游戏、桌面
- **原理**：软件抓屏取色 → 输出到灯带/控制器
- **代价**：对主机/外接 HDMI 设备不天然通用

### 路线 B：外部 HDMI 采集（可覆盖主机/机顶盒）

- **适合**：主机、电视盒子、任何 HDMI 输入
- **原理**：HDMI 外部采集 → 控制器驱动 LED
- **代价**：硬件更多、延迟与兼容性变量更大、走线更复杂

## 常见 DIY 硬件组合（WS2812 + 微控制器）

入门最常见的堆叠：

- **灯带**：WS2812B / SK6812（可寻址）
- **控制器**：ESP32（Wi‑Fi、性能强）或 Arduino（简单、稳定）
- **供电**：独立 5V 电源（按灯珠数量留裕量）
- **可选**：电平转换（长线/干扰环境下更稳）

想看更多实体装机思路，Instructables 上有大量“显示器背光/环境光”项目（示例搜索： [Instructables — ambient light monitor](https://www.instructables.com/howto/ambient+light+monitor/)）。

## 核心步骤（按这个顺序最少返工）

1. **先量尺寸，规划 LED 布局**
   - 方向：顺时针还是逆时针
   - 起点：LED #0 从哪一角开始（常见是左下或右下）
2. **确定取色方式**
   - PC 软件直抓（更简单）vs 外部 HDMI 采集（更通用）
3. **把供电做对**
   - 长灯带不要指望 USB 供电
   - 长距离考虑“多点注入供电”减少压降导致的偏色
4. **刷写/配置控制器固件**
5. **做分区映射（软件分区 ↔ 物理 LED）**
6. **调平滑/亮度/白平衡**

## 成本预估（粗略）

主要由 LED 密度与做工决定：

- 灯带 + 扩散：低到中
- ESP32 + 线材 + 接头：低
- 电源与安全冗余：中

## 外部采集 vs 软件直抓：真正影响的点

- **延迟**：外部链路更容易引入延迟；PC 直抓可做到很快
- **稳定性**：外部方案“输入通吃”，但故障点更多
- **时间成本**：外部采集需要更多物理走线与参数适配（甚至涉及 EDID/HDCP）
